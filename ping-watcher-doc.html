◉ PingWatch

API monitoring that alerts you before your customers do.

Vue 3 Supabase Tailwind v4 MIT License

PingWatch is a full-stack SaaS application that monitors your API endpoints, detects downtime, sends alerts via email and Slack, and lets you share beautiful public status pages with your customers.

💡
Just getting started? Jump to Installation to get the app running locally in under 5 minutes.

✨ Features

⚡
Real-time Monitoring
Ping any HTTP/HTTPS endpoint every 30 seconds to 5 minutes depending on your plan.
🔔
Instant Alerts
Email and Slack notifications the second a monitor goes down or recovers.
◫
Public Status Pages
Shareable uptime pages at /status/your-slug with 90-day history bars.
📊
Analytics
24h / 7d / 30d uptime percentages, latency trends, incident distribution charts.
⚠
Incident Timeline
Full incident history with auto-calculated downtime duration and resolution tracking.
💳
Subscription Billing
Lemon Squeezy integration with Free / Starter / Pro plans and automatic plan enforcement.

🔑 Environment Variables

Create a .env file in your project root:

# Supabase — find these in Dashboard → Project Settings → API
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here

# Lemon Squeezy — find Variant IDs in your LS product page
VITE_LS_STARTER_VARIANT_ID=123456
VITE_LS_PRO_VARIANT_ID=789012
VITE_LS_STORE_SLUG=your-store-slug
⚠
Never commit your .env file. It's already in .gitignore by default with Vite.

Supabase Edge Function Secrets

These are set via the Supabase CLI — not in your .env file:

supabase secrets set RESEND_API_KEY=re_your_key
supabase secrets set RESEND_FROM=PingWatch <alerts@yourdomain.com>
supabase secrets set LEMONSQUEEZY_WEBHOOK_SECRET=your_secret
supabase secrets set LEMONSQUEEZY_API_KEY=your_api_key

🗄 Database Setup

All tables, RLS policies, and functions are in pingwatch_schema.sql.

Step 1 — Enable extensions

In Supabase Dashboard → Database → Extensions, enable:

  • pg_cron — scheduled monitor checks
  • pg_net — HTTP calls from scheduled jobs
  • pgcrypto — UUID generation (usually already enabled)

Step 2 — Run the schema

Go to Supabase Dashboard → SQL Editor → New Query, paste the entire contents of pingwatch_schema.sql and click Run.

ℹ
The schema creates 8 tables, 10 RLS policies, 4 functions, 2 triggers, and 2 pg_cron jobs automatically.

⚙ Edge Functions

PingWatch uses 3 Supabase Edge Functions (Deno runtime):

FunctionTriggerPurpose
run-checkspg_cron every minutePing all due monitors, detect incidents, fire alerts
send-alertCalled by run-checksSend email (Resend) + Slack notifications
ls-webhookLemon Squeezy webhookSync subscription plan changes to profiles table

Deploy all functions

npm install -g supabase
supabase login
supabase link --project-ref YOUR_PROJECT_REF

supabase functions deploy run-checks
supabase functions deploy send-alert
supabase functions deploy ls-webhook

⏱ Scheduled Jobs

After deploying Edge Functions, update the pg_cron job to point to your production URL.

Run this in Supabase SQL Editor:

-- Remove old job
SELECT cron.unschedule('run-monitor-checks');

-- Schedule with production URL
SELECT cron.schedule(
  'run-monitor-checks',
  '* * * * *',
  $$
    SELECT net.http_post(
      url     := 'https://YOUR_PROJECT.supabase.co/functions/v1/run-checks',
      headers := jsonb_build_object(
        'Authorization', 'Bearer YOUR_SERVICE_ROLE_KEY'
      ),
      body    := '{}'::jsonb
    );
  $$
);

-- Verify
SELECT * FROM cron.job;
⚠
Replace YOUR_PROJECT and YOUR_SERVICE_ROLE_KEY with your real values from Supabase Dashboard → Project Settings → API.
